Your secrets are safe with us.

Encryption

All secrets are encrypted using AES-256-GCM before being stored. We use a hierarchical key structure where each project has its own encryption key, which is itself encrypted with a master key stored in a hardware security module (HSM).

  • AES-256-GCM encryption at rest
  • TLS 1.3 encryption in transit
  • Per-project encryption keys
  • Hardware security modules (HSM) for master key storage
  • Regular key rotation (optional)

Zero-Knowledge Architecture

We've designed Vault so that we cannot access your secrets—even if we wanted to. All encryption and decryption happens client-side, meaning your plaintext secrets never touch our servers.

Your secrets are encrypted before leaving your device using a key derived from your account credentials. We only ever see and store the encrypted ciphertext. This means:

  • Vault employees cannot view your secrets
  • A database breach would not expose plaintext secrets
  • Government requests cannot compel us to reveal data we don't have

Infrastructure Security

Vault runs on hardened infrastructure with multiple layers of protection.

  • Hosted on AWS with SOC 2 certified data centers
  • Private VPC with no public-facing database access
  • Web Application Firewall (WAF) protection
  • DDoS mitigation at the edge
  • Regular penetration testing by third-party security firms
  • 24/7 intrusion detection and monitoring

Access Controls

Fine-grained permissions ensure team members only have access to what they need.

  • Role-based access control (RBAC)
  • Environment-level permissions (e.g., production read-only)
  • IP allowlisting for API access
  • Mandatory 2FA for team owners
  • SSO/SAML integration with identity providers
  • Automatic session expiration

Compliance

Vault meets the compliance requirements of security-conscious organizations.

  • SOC 2 Type II: Independently audited annually
  • GDPR: EU data residency, DPA available
  • HIPAA: BAA available for Enterprise customers
  • CCPA: California privacy requirements met

Responsible Disclosure

We take security vulnerabilities seriously. If you discover a security issue, please report it to us responsibly:

  • Email: security@vault.dev
  • PGP key available for encrypted communications
  • We aim to respond within 24 hours
  • No legal action for good-faith research

We offer a bug bounty program for qualifying vulnerabilities. Rewards range from $100 to $10,000 depending on severity.
