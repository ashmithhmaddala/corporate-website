API Reference

The Vault API is a REST API that allows you to manage projects, secrets, environments, and team members programmatically. All requests must be authenticated with an API token.

Base URL

https://api.vault.dev/v1

Authentication

All API requests require a bearer token in the Authorization header:

Authorization: Bearer vlt_your_api_token_here

You can generate API tokens from your dashboard under Settings → API Tokens. Tokens can be scoped to specific projects and actions.

Rate Limits

The API is rate limited to 100 requests per minute per token. Rate limit headers are included in every response:

X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1706789400

Endpoints

GET/projects

List all projects you have access to.

Response

{
  "data": [
    {
      "id": "proj_abc123",
      "name": "my-api",
      "created_at": "2025-12-01T10:00:00Z",
      "environments": ["development", "staging", "production"]
    }
  ],
  "meta": {
    "total": 1,
    "page": 1,
    "per_page": 20
  }
}

POST/projects

Create a new project.

Request Body

{
  "name": "my-new-project",
  "description": "Optional project description"
}

Response

{
  "data": {
    "id": "proj_xyz789",
    "name": "my-new-project",
    "description": "Optional project description",
    "created_at": "2026-01-30T15:30:00Z",
    "environments": ["development", "staging", "production"]
  }
}

GET/projects/:id/secrets

List all secrets in a project for a specific environment.

Query Parameters

  • environment (required) – The environment name (e.g., "production")

Response

{
  "data": [
    {
      "key": "DATABASE_URL",
      "value": "postgres://user:pass@host:5432/db",
      "updated_at": "2026-01-28T12:00:00Z",
      "updated_by": "user_abc123"
    },
    {
      "key": "STRIPE_SECRET_KEY",
      "value": "sk_live_...",
      "updated_at": "2026-01-15T09:30:00Z",
      "updated_by": "user_abc123"
    }
  ]
}

POST/projects/:id/secrets

Create or update a secret.

Request Body

{
  "environment": "production",
  "key": "NEW_SECRET",
  "value": "secret_value_here"
}

Response

{
  "data": {
    "key": "NEW_SECRET",
    "environment": "production",
    "created_at": "2026-01-30T16:00:00Z"
  }
}

DELETE/projects/:id/secrets/:key

Delete a secret from an environment.

Query Parameters

  • environment (required) – The environment name

Response

{
  "data": {
    "deleted": true
  }
}

Errors

The API uses standard HTTP status codes. Errors include a JSON body with details:

{
  "error": {
    "code": "unauthorized",
    "message": "Invalid API token",
    "status": 401
  }
}

Common Error Codes

  • 400 – Bad Request: Invalid parameters
  • 401 – Unauthorized: Invalid or missing token
  • 403 – Forbidden: Token doesn't have required permissions
  • 404 – Not Found: Resource doesn't exist
  • 429 – Too Many Requests: Rate limit exceeded
  • 500 – Internal Server Error: Something went wrong on our end

SDKs

We provide official SDKs for common languages:

  • Node.js SDK – npm install @vault/sdk
  • Python SDK – pip install vault-sdk
  • Go SDK – go get github.com/vault/vault-go
  • Ruby SDK – gem install vault-sdk
