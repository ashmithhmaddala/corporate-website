Introduction

Vault is secrets infrastructure for modern engineering teams. It provides a secure, centralized place to store and manage all your application secrets—API keys, database credentials, certificates, and more.

Why Vault?

Managing secrets is hard. Teams typically start by sharing credentials over Slack, then graduate to a shared password manager, then maybe to environment files checked into git (bad idea). Each approach has problems:

  • Slack/Email: No access control, no audit trail, credentials sit in chat history forever
  • Password managers: Not designed for programmatic access, no environment separation
  • .env files in git: Credentials exposed to anyone with repo access, impossible to rotate
  • Platform-specific solutions: Lock-in, different interfaces for each platform

Vault solves these problems by giving you one place to manage secrets, with proper access controls, audit logging, and integrations that sync secrets to wherever they need to go.

Core Concepts

Projects

A project represents an application or service. Each project has its own set of secrets, environments, and team members. You might have one project for your API, another for your web app, and another for your mobile app.

Environments

Environments represent deployment targets like development, staging, and production. Each environment has its own copy of secrets with potentially different values. A developer can have access to development secrets without being able to see production credentials.

Secrets

Secrets are key-value pairs like DATABASE_URL or STRIPE_SECRET_KEY. Each secret exists in every environment but can have different values. Secrets are encrypted at rest and in transit.

Quick Start

The fastest way to get started is with our CLI:

# Install the CLI
npm install -g @vault/cli

# Login to your account
vault login

# Create a new project
vault init my-project

# Add a secret
vault set DATABASE_URL "postgres://..."

# Run your app with secrets injected
vault run -- npm start

That's it! Your secrets are now managed by Vault. No more .env files, no more copy-pasting credentials.
